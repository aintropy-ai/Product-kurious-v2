1
Manager or Admin Initiates Offboarding
TRIGGER
Someone says "Offboard [name]" in Kuri. Agent first checks who is making the request before proceeding.
Permission check:
IF reporting manager → allow and proceed
IF admin → allow and proceed
IF neither → deny: "Only [name]'s reporting manager or an admin can initiate offboarding."
Data collected:
Employee name, last working day, reason for departure (resignation, termination, layoff, contract end, internship end), notice period
IF all details provided → proceed to Step 2
IF any detail missing → ask manager for missing info
IF contractor with contract end date → agent may auto-detect and suggest
2
Classify Offboarding & Generate Plan
AUTOMATIC
This is the foundational step. The agent classifies the departure and generates a tailored offboarding plan. Every step that follows is shaped by this classification.
Agent checks three things:
A. Employment type: Full-time, Contractor, Intern, Advisor
B. Reason for departure: Resignation, Termination, Layoff, Contract end, Internship end
C. Jurisdiction: Country, state/province, company policy, legal requirements

3
Calculate Timeline
AUTOMATIC
Agent calculates the full offboarding timeline based on classification from Step 2.
IF last day is today or passed → URGENT. All steps execute immediately.
IF termination → skip timeline, execute critical steps immediately
IF last day > 30 days away → weekly check-in cadence
IF last day < 2 weeks away → start all steps immediately
4
Notify Employee
AUTOMATIC
Agent sends personalized message. Tone determined by classification.
IF resignation → supportive: "We're sorry to see you go..."
IF termination → professional: "Your employment ends on [date]..."
IF layoff → empathetic: "We want to make this transition smooth..."
IF contract end → neutral: "Your contract ends on [date]..."
IF internship end → warm: "Your internship wraps up on [date]. Thank you..."
5
Exit Interview
AUTOMATIC
Applies to: Resignation, Layoff, Intern end. Optional: Termination, Contractor. N/A: Advisor.
IF applicable per matrix → schedule from calendars, send pre-survey
IF employee declines → log as declined, don't force
IF 3+ exits mention same issue → flag pattern to admin
6
Exit Documents
NEEDS APPROVAL
Agent drafts exit documents based on classification. Admin must review before sending.
Documents by classification:
FT Resignation: NDA reaffirm, non-compete, IP assignment, separation ack, PTO calc
FT Termination: Separation + release, NDA, IP, immediate pay, COBRA
FT Layoff: Severance + release, NDA, WARN Act, benefits continuation
Contractor: Contract closeout, deliverable sign-off, final invoice
Intern: NDA reminder, simple separation ack
Advisor: Equity closeout, confidentiality reminder
IF admin approves → send to employee for signature
IF employee signs → stored, proceed
IF no sign in 48hrs → reminder
IF refuses → flag to admin and legal
7
Exit Checklist for Employee
AUTOMATIC
Personalized checklist based on classification. Full-time gets full list, contractor gets deliverable + invoice focus, intern gets simplified version.
IF item not done 3 days before last day → remind employee
IF item not done on last day → escalate to manager
8
Automated Work Handoff
AUTOMATIC
Fully automated. Agent scans all connected tools and reassigns all in-progress work. No manual handoff required.
Code & Engineering:
Scan GitHub for open PRs and assigned issues → auto-reassign to manager. Flag branches with uncommitted work.
Project Management:
Scan Trello/Jira/Notion for assigned tasks → auto-reassign. Flag tasks where person is sole assignee.
Calendar & Meetings:
Transfer organized meetings to manager. Flag external client meetings for re-introduction.
Documents & Files:
Scan Drive/Notion for sole-owner files → auto-transfer ownership. Ensure shared folders have other admins.
Communication:
Set up email auto-reply. Configure forwarding to manager (if admin approves).
Agent generates handoff summary
"Here's everything [name] had in progress. I've automatically reassigned it all to [manager]: 3 open PRs, 7 tasks, 2 client meetings flagged, 5 sole-owner documents transferred. If you want to reassign anything, just tell me."
IF resignation/layoff/contract end → handoff runs during notice period
IF termination → handoff runs immediately, same day
9
Team Communication
AUTOMATIC / MANAGER-CONTROLLED
Routine departures: agent auto-sends (manager can edit). Sensitive departures: agent drafts, NEVER auto-sends.
Auto-notify (manager can edit before send):
Resignation: "[Name] is leaving on [date]. We wish them well."
Contract end: "[Name]'s contract concludes on [date]."
Internship end: "[Name]'s internship wraps up on [date]. Thank you!"
Manager-controlled (agent drafts, waits for explicit approval):
Termination: Agent drafts but NEVER sends. Manager decides timing and wording.
Layoff: Same. Agent drafts, manager controls everything.
If departing employee managed people:
Agent identifies direct reports → asks manager who they should report to → notifies direct reports privately before any team announcement.
External contacts/clients:
Agent scans for client meetings/CRM entries → flags to manager for re-introduction. Agent does NOT contact external parties.
10
Schedule Access Revocation
AUTOMATIC
Agent categorizes all tools and schedules revocation. Timing based on classification.
Non-sensitive (auto-revoke, no approval):
Slack, Google Calendar, Discord, Notion, shared drives
Sensitive (requires admin approval):
GitHub, Claude Code, AWS/GCP, admin panels, financial tools, production systems
IF resignation/layoff/contract/intern → non-sensitive on last day, sensitive approval 5 days before
IF termination → ALL revocation requested immediately. URGENT.
IF last day passed and access still active → daily URGENT alert: security risk
11
Admin Approves Sensitive Revocation
NEEDS APPROVAL
Agent presents each sensitive tool with details. Options: [Revoke all now] [Revoke on last day] [Review individually]
IF no response 24hrs → reminder
IF no response 48hrs → escalate to super admin
IF termination and no response 4hrs → escalate immediately (security risk)
12
Calculate Final Payment
AUTOMATIC
Agent calculates based on classification. Includes jurisdiction-specific rules.
FT Resignation: Prorated salary + PTO payout + expenses - deductions
FT Termination: Immediate pay + severance (if applicable) + PTO + COBRA
FT Layoff: Severance package + prorated salary + PTO + benefits
Contractor: Final invoice per contract terms + expenses
Intern: Prorated stipend
Advisor: Per advisory agreement + equity vesting status
Jurisdiction flags
California: same-day final pay on termination. Other states: varies. India: gratuity if 5+ years. Canada: statutory severance. Agent flags timing requirements.
13
Admin Approves Final Payment
NEEDS APPROVAL
Full breakdown shown. Agents NEVER release payment without admin approval. Options: [Approve] [Adjust] [Hold]
IF approved → proceed to execution
IF adjusted → recalculate, re-present
IF held → ask when to revisit
IF termination + jurisdiction requires immediate pay → URGENT reminders from hour 1
14
Execute Revocations and Payment
AUTOMATIC
Revoke each tool, verify via API, log with timestamp. Initiate ACH payment. Notify employee and admin.
IF revocation fails → retry once, then alert admin for manual revocation
15
Equipment Return
AUTOMATIC
Applies to: Full-time (all), Interns, Contractors (if equipment issued). N/A: Advisors.
IF 3 days before last day → send reminder with specific item list
IF not returned by last day → daily reminder
IF not returned 7 days after → escalate to admin
IF termination → equipment return flagged as urgent from day 1
IF remote allowance (no physical equipment) → mark N/A
16
Benefits & Insurance Cutoff Review
PLACEHOLDER
Applies to: Full-time employees only. Currently a skeleton step — creates a manual review task. Full automation to be built later.
What the agent does now:
Check if employee was enrolled in benefits (health, dental, vision, life, 401k).
Calculate benefit cutoff date based on last day and company policy.
Create task for admin: "Review benefits cutoff for [name]. Insurance ends [date]. Check COBRA/equivalent."
To be built later (with Nirmit):
Automatic COBRA notice generation (US). Benefits continuation calculation. 401k rollover instructions. State-specific rules. Integration with benefits providers.
17
Rehire Eligibility
AUTOMATIC
Agent auto-sets rehire eligibility based on classification. Manager can override.
IF resignation → Eligible
IF termination → Not eligible (manager can override)
IF layoff → Eligible
IF contract end → Eligible for future contracts
IF internship end → Eligible (flag for return offer consideration)
18
Create Alumni Record
AUTOMATIC
Permanent record: name, role, tenure, exit notes, rehire status, key contributions.
IF resignation/layoff/contract/intern → always create
IF termination → manager decides whether to create
IF employee opts into alumni network → add to directory
IF opts out → internal record only
19
Final Day IT/Admin Checklist
PROACTIVE
On last day, agent runs a final automated verification of everything.
Verification checklist:
□ Email auto-reply set up and forwarding configured
□ All non-sensitive tool access revoked
□ All sensitive tool access revoked (with approval)
□ All in-progress work reassigned
□ All sole-owner documents transferred
□ Equipment return initiated or completed
□ Final payment approved and initiated
□ Exit documents signed
□ Benefits cutoff review task created
□ Team notification sent (if applicable)
□ Alumni record created
End of day report to admin
"End of day check for [name]'s offboarding: Completed: [X] items. Pending: [list]. Action needed: [list]."
✓
Offboarding Complete
COMPLETE
Agent verifies all success criteria are met based on classification.
Notifications:
To admin: "Offboarding complete. Classification: [type]/[reason]. All access revoked, payment processed, work reassigned to [manager]. [X] items auto, [X] needed approval."

To manager: "[Name]'s offboarding complete. All work reassigned to you. [X PRs, X tasks, X docs transferred]."

To employee (resignation/contract/intern): "Thank you for your time at AIntropy. Your final payment is on its way. We wish you all the best."
To employee (termination/layoff): Professional, factual closing message only.

Error Handling
EDGE CASES
Contract end auto-detected
Agent detects end date approaching → alerts admin: "Start offboarding?"
Employee refuses exit checklist
Log refusal, notify manager. Agent does not force.
Revocation fails
Retry once. If fails, alert admin for manual revocation.
Equipment not returned 14 days
Escalate to admin. Options: deduct, formal notice, or write off.
Payment disputed
Pause payment, flag to admin and finance for review.
Employee rehired during offboarding
Halt all steps. Ask admin: cancel offboarding or complete and re-onboard?
Manager leaves during offboarding
Alert admin to reassign. Pause manager-dependent steps.
Departing employee has direct reports
Flag to manager for reassignment before team notification.
Work handoff finds sensitive files
Flag to admin instead of auto-transferring.
