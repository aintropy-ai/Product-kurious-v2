Workflow

Offer Letter Drafting Workflow

From "draft an offer" to signed contract triggering onboarding

1
Admin Requests Offer Letter TRIGGER
Admin says "Draft an offer for [name]" in Kuri, or a new hire record is created in the HR tool.
2
Gather Context AUTO
Agent gathers required information or asks admin for anything missing. Needs: name, role, employment type, country/state, compensation, start date, benefits eligibility.
3
Select Template by Type & Jurisdiction AUTO
Agent selects the correct legal template based on employment type and jurisdiction.
IF US full-time → at-will employment, benefits, IP assignment.
IF US contractor → scope of work, payment terms, no benefits.
IF India → notice period, gratuity, provident fund.
IF intern → stipend, duration, learning objectives.
IF advisor → equity terms, confidentiality, time commitment.
4
Draft Offer Letter with Legal Clauses AUTO
Agent drafts the complete offer letter with all required legal clauses: NDA, non-compete (where enforceable), IP assignment, compensation, benefits, and jurisdiction-specific language.
5
Admin Reviews & Edits APPROVAL NEEDED
Admin reviews the drafted offer letter and makes any edits before sending.
Legal documents are NEVER auto-sent. Always requires human review and explicit approval.
6
Sent for E-Signature AUTO
After admin approval, the offer letter is sent to the candidate for electronic signature.
7
Signed Contract Stored AUTO
Signed contract is securely stored with full version history for compliance and audit purposes.
8
Triggers Onboarding Workflow AUTO
Contract signing automatically triggers the Onboarding Workflow (S1). No manual handoff needed.

Right template. Right clauses. Right jurisdiction. Admin reviews. Onboarding starts automatically.
