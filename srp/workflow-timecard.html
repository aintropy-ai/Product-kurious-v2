Time Card Curator Workflow

Auto-fill first, ask second. No clock-in, no clock-out. The agent handles everything.

Core Principle
1. Contract terms = baseline. Offer says 9-6 M-F? Timesheet is pre-filled from day 1.
2. Any tool activity = validation. Discord, Docs, GitHub, Calendar — any signal from any tool confirms the work day.
3. No signal = question. Only when zero activity across all tools does the agent ask.
4. Exceptions get flagged. Overtime, missing days, unusual hours go to manager. Everything else is auto-approved.

1
Initialize from Onboarding Data
TRIGGER — DAY 1
When a new employee is onboarded, agent auto-creates their timesheet profile by pulling data from the onboarding agent.
Data pulled automatically:
Employment type, working hours from contract, work days, time zone, start date, pay type (salaried/hourly), hourly rate (if applicable), manager, PTO allocation
IF all data present → profile created, baseline set from day 1. Zero employee effort.
IF contract says "flexible" → monitor signals for 2 weeks to find pattern, then set as baseline
IF contractor with no fixed hours → no baseline, track from tool activity
IF data missing → ask admin: "What are [name]'s working hours?"
2
Daily Activity Signal Collection
AUTOMATIC — DAILY
Agent passively collects activity signals from all connected tools. Not surveillance — no screenshots, no keystroke logging, no content reading. Only checks: was there activity or not?
Any signal from any tool = work day confirmed
Discord/Slack (message, reaction, voice channel) • Google Calendar (meeting attended) • GitHub (commit, PR, review, comment) • Google Docs/Notion (edit, comment) • Email (sent an email) • Jira/Trello/Linear (task update) • SSO login to any tool • Any other registered tool
Important rules
A Discord message is as valid as a Git commit. Not everyone uses every tool. A marketing person on Discord + Docs is treated the same as an engineer on GitHub.

Agent does NOT read content. Does NOT rank tools. Does NOT track exact hours for salaried employees — just confirms "worked today."

For contractors (hourly): Agent tracks active time window (first signal to last signal) as an estimate for timesheet drafting.
3
Daily Timesheet Entry
AUTOMATIC — DAILY
At end of each day, agent creates or confirms the timesheet entry. No employee action required for salaried workers.
Salaried employees:
IF activity detected on any tool → auto-fill baseline hours (e.g., 8 hrs). No question. No notification.
IF no activity on a scheduled work day → flag for Missing Day Detection (Step 4)
IF activity on a non-scheduled day → flag for Overtime Detection (Step 5)

Contractors (hourly):
IF activity detected → draft entry with estimated hours (first signal to last signal)
IF no activity → no entry (contractors only paid for tracked hours)

Founders:
→ auto-fill standard hours every work day. No signals checked.
4
Missing Day Detection
AUTOMATIC
Triggered when: scheduled work day, but zero activity detected across ALL tools.
Agent asks employee in Kuri
"Hey [name], I didn't see any activity on [day]. No worries — just checking. Was that PTO, sick day, or were you working offline?"

Options: PTO • Sick day • Working offline • Public holiday • Personal day (unpaid)
IF employee responds → log accordingly, update timesheet
IF "working offline" → mark as work day. Agent learns this person works offline sometimes.
IF no response 24hrs → gentle reminder
IF no response 48hrs → notify manager: "[Name] has an unaccounted day."
IF 3+ offline days/month → agent adapts: "You often work offline on [day]. Auto-mark as work day going forward?"
5
Overtime Detection
PROACTIVE
Triggered when activity is detected outside baseline schedule: after hours, weekends, or weekly total exceeds contract hours.
Agent asks employee first:
"I noticed you were active on [Saturday / until 11pm]. Want me to log this as overtime?"

IF employee says yes → log as overtime, auto-calculate rate (1.5x or 2x per jurisdiction), flag to manager
IF employee says "no, just checking something" → don't log. No further action.

Compliance auto-checks:
IF weekly hours exceed state maximum → ALERT to manager
IF 7+ consecutive work days → flag rest day requirement
IF overtime 3+ consecutive weeks → flag: "Possible workload issue"
6
PTO Integration
AUTOMATIC
PTO balance tracked alongside timesheets. Employees request PTO through Kuri.
PTO request flow
Employee: "I want to take Friday off"
Agent checks balance → sends to manager → manager approves → timesheet updated → balance deducted
IF enough PTO balance → send approval to manager
IF no balance → "You don't have enough PTO. Want to request an unpaid day instead?"
IF public holiday → auto-marked, no PTO deduction, everyone notified
IF employee works during approved PTO → flag: "Activity detected on your PTO day. Was this intentional?"
7
Weekly Timesheet Summary
EVERY FRIDAY
Every Friday, agent sends each employee their weekly summary via Kuri.
Employee sees
"Here's your week:
Mon: 8 hrs ✓ • Tue: 8 hrs ✓ • Wed: PTO (approved) • Thu: 8 hrs + 2 OT (pending) • Fri: 8 hrs ✓
Total: 34 regular + 2 overtime. Looks right?"
IF employee confirms → finalized
IF employee corrects → adjust and re-confirm
IF no response by Sunday EOD (salaried) → auto-finalize based on signals
IF no response by Sunday EOD (contractor) → urgent reminder Monday morning
8
Manager Approval
AUTO-APPROVE / EXCEPTIONS ONLY
Normal weeks are auto-approved. Manager only sees exceptions.
Auto-approved (no manager action):
Regular work weeks matching baseline. No overtime. No missing days. PTO already approved earlier.
Flagged for manager:
Overtime hours (with calculated cost). Unaccounted missing days. All contractor timesheets. Employee corrections. First week of new employee.
Manager sees on Friday/Monday
"Weekly review: 5 employees, 4 auto-approved. 1 needs attention: Alex — 6 hrs overtime ($[cost]). [Approve] [Reject] [Adjust]"
IF no response Monday EOD → reminder
IF no response Tuesday EOD → escalate to admin
9
Payroll Handoff
AUTOMATIC
Approved timesheets flow directly to the Auto Payroll Processor agent. No export, no manual entry.
Data passed:
Regular hours, overtime hours (with rate), PTO days, sick days, unpaid days, contractor hours, adjustments
IF all timesheets approved → handoff to payroll: "All timesheets ready."
IF some pending → "Payroll waiting on approval for: [list]"
IF disputed → hold that employee, process everyone else
10
Compliance Monitoring
PROACTIVE — CONTINUOUS
Agent continuously monitors for labor law compliance. Knows state-specific rules so the founder doesn't have to.
Weekly overtime: Auto-calculate after 40hrs/week (or state threshold). Alert when approaching.
Daily overtime: California = after 8hrs/day. Agent tracks per state.
Rest days: Flag 7+ consecutive work days.
Meal breaks: California = 30min after 5hrs. Agent flags when not taken.
PTO payout laws: Some states require payout on termination. Agent tracks accrued balance.
Minimum wage: Verify hourly rate meets federal + state minimum.
Record keeping: Maintain records for 3+ years (or per jurisdiction).
Proactive alerts
"[Name] is at 38 hours. Overtime kicks in after 40."
"[Name] has worked 6 consecutive days. Rest day required."
"[Name] hasn't taken a meal break in 5.5 hours."
11
Schedule Adaptation
LEARNS OVER TIME
After 4 weeks, agent compares actual patterns to contract baseline and suggests adjustments.
IF pattern consistently differs (e.g., works 10-7 not 9-6) → "I've noticed you usually work 10am-7pm. Want me to update your schedule?"
IF employee says yes → baseline updated for future
IF employee says no → keep original
IF employee frequently works offline → "Auto-mark [day] as work day going forward?"
IF flexible schedule → no forced baseline, use signals + weekly confirmation
12
Timesheet History & Audit Trail
AUTOMATIC
Every action logged and searchable. Every entry, how it was created, who approved it, all corrections.
Accessible to:
Employee: own timesheets, PTO balance, history
Manager: team timesheets, approvals, overtime trends
Admin: everyone's timesheets, compliance records, full audit trail
13
Notifications & Reminders
AUTOMATIC
Missing day: employee, same day or next morning
Overtime detected: employee then manager
Weekly summary: employee, Friday EOD
Approval needed: manager, Friday/Monday
PTO request: manager, immediately
PTO decision: employee, immediately
Approaching overtime: manager, at 90% threshold
Compliance violation: manager + admin, immediately
Payroll waiting: manager, before deadline
PTO balance low: employee, below 2 days
14
Timesheet Corrections
EMPLOYEE / MANAGER
Both employees and managers can correct timesheets at any time. All changes logged.
Employee correction:
IF week already approved → flag to manager for re-approval
IF correction affects processed payroll → "Adjustment will apply in next payroll run"
IF correction older than 30 days → requires admin approval

Manager correction:
→ update entry, notify employee of change, log in audit trail
Error States
!
Error Handling
EDGE CASES
No tools connected
Alert admin: "No tools connected for [name]. Can't verify activity. Connect at least one tool or switch to manual entry."
Tool API goes down
Don't mark as missing day. Use other sources. If all down, assume work day and note the outage.
Employee disputes entry
Allow correction, log it, re-route for approval if already approved.
Contractor doesn't confirm timesheet
Block their payroll entry. Remind daily. After 3 days, notify manager.
PTO request on a day with hours logged
"You already have hours for [date]. Replace with PTO?"
Retroactive change after payroll
Queue adjustment for next payroll cycle. Don't reprocess current cycle.
