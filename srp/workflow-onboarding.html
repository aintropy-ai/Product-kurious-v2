1
Contract Signed
TRIGGER
Contract signed on any connected HR tool (Gusto, BambooHR, or any registered source). Agent automatically detects and extracts all details.
Data extracted:
Full name, personal email, role and title, employment type (full-time, contractor, intern, advisor), country and state, start date, reporting manager, compensation details, work type (in-office, remote, hybrid)
IF all required fields present → proceed to Step 2
IF any field missing → ask the user to provide the full details: "Alex Chen's contract is signed but I'm missing [field]. Can you provide this before I start onboarding?"
IF user doesn't respond in 4 hours → reminder
IF user doesn't respond in 24 hours → escalate to super admin
2
Create Company Email
AUTOMATIC
Agent creates email account: firstname.lastname@company.com. All employment types (full-time, contractor, intern) get full Google Workspace license.
IF email available → create account
IF email taken → try alternatives: firstname.middleinitial@, firstname.lastname2@
IF no alternative works → ask user: "Standard email formats are taken for Alex Chen. What email should I use?"
On success
Log: "Google Workspace created, alex.chen@aintropy.ai, 6:00 AM"
Welcome email sent to personal email with login instructions.
ON FAILURE: Retry once with alternate config. If still fails, alert user with specific error. Do NOT block remaining steps, continue provisioning other tools in parallel.
3
Send Welcome Message via Kuri
AUTOMATIC
Agent sends a personalized welcome message showing what's already been set up and what's still in progress.
IF employee has logged in → show in Kuri directly
IF not logged in yet → send welcome email with link to Kuri

IF all tools succeeded → show all green checkmarks
IF any tool failed → show honestly: "I'm still working on [tool]. I'll let you know when it's ready."
Welcome message content
"Welcome to AIntropy, Alex! 🎉 I'm your onboarding assistant. While you were signing your contract, I've already set up everything you need:

✓ Google Workspace: alex.chen@aintropy.ai
✓ GitHub: added to aintropy-ai org
✓ Discord: invite sent
⏳ Claude Code: provisioning in progress

Let's complete your profile now. It takes about 2 minutes."
4
Policy Acknowledgement
AUTOMATIC
This is a GATE step. No internal tools (GitHub, Discord, Slack, etc.) will be provisioned until the new hire has acknowledged all required policies.
Policies sent:
Company handbook, NDA, code of conduct, security policy, acceptable use policy. List is configurable by admin.
Agent: "Before I give you access to our tools, I need you to review and acknowledge a few company policies. This is standard for all new hires."

IF employee acknowledges all → proceed to tool provisioning (Step 5)
IF employee hasn't acknowledged within 24 hrs → reminder
IF employee hasn't acknowledged within 48 hrs → escalate to admin: "Alex hasn't signed the required policies. Tool access is on hold."
5
Provision GitHub Access
AUTOMATIC
Gated by Step 4. Only proceeds after policy acknowledgement.

Agent adds user to GitHub organization with team assignment based on role.
IF role is engineering, technical, or intern → provision GitHub
IF role is HR, finance, marketing, advisor → skip this step

Team assignment:
IF engineer → engineering team, all eng repos
IF intern → intern team, limited repos
IF contractor → contractor team, read-only on sensitive repos
ON FAILURE: Check if email is already linked to another GitHub account. If yes, send invite to existing account. If no, retry once. If still fails, alert user: "GitHub invite failed. Their email may already be linked to another org."
Continue to next step regardless.
6
Provision Discord
AUTOMATIC
Gated by Step 4. Only proceeds after policy acknowledgement.

Everyone gets Discord with full access regardless of employment type. All channels, full member role. No restrictions.
Admin notification
"I've given Alex full access to Discord (all channels, full member role). If you want to adjust their channels or permissions, just let me know."
ON FAILURE: Retry once. If fails, alert admin. Continue to next step.
7
Provision All Other Registered Tools
AUTOMATIC
Gated by Step 4. Only proceeds after policy acknowledgement.

Agent checks the data source registry for all other tools (Slack, Claude Code, Cloudflare, HuggingFace, Notion, Jira, etc.). Not hardcoded. When admin adds a new tool tomorrow, the agent automatically includes it.
FOR EACH registered tool:
IF role-tool mapping says this role needs it → provision
IF role-tool mapping says skip → skip
IF not configured for this role → ask admin: "Alex Chen (intern) needs access to [tool]. Should I provision it?"
Admin notification after all tools provisioned
"Alex has signed all required policies. I've now provisioned full access to Google Workspace, GitHub, Discord, and all other registered tools. If you want to adjust any permissions, let me know."
8
Create Employee Profile
AUTOMATIC
Agent creates profile in team directory: name, role, title, start date, reporting manager, team, location, work type, employment type. Profile marked as "Incomplete" until employee finishes setup.
IF photo provided in HR tool → auto-populate
IF not → ask during guided setup (optional, no reminder)
On success
Profile visible to team with "New hire" badge. This step always succeeds (internal system).
9
Guided Profile Completion
CONVERSATION
Kuri guides the new hire through completing their profile, step by step in conversation.
9a: Profile Photo
"First, let's add a profile photo so the team knows who you are."
IF uploads → store, propagate to directory and all views
IF skips → no reminder, completely optional. Proceed to next.
9b: Birthday
"When's your birthday? This is for HR records and team celebrations."
IF provides date → validate format, store
IF invalid format → "I didn't catch that. Can you try again? (e.g. April 15)"
IF skips → store as "not provided", optional field
9c: Emergency Contact ⚠️ Required
"Important one. We need an emergency contact for safety. Name, phone number, and relationship."
IF provides all three → validate phone, store (marked confidential)
IF partial → "I got [name]. Can you also share their phone number and relationship?"
IF tries to skip → "This one is required for safety. I can't skip it."
Company policy configurable: IF mandatory → remind daily. IF optional → remind in 48 hrs.
9d: Fun Facts
"Last one! What's your superpower? Favorite snack? Hidden talent?"
IF answers → store, display on profile
IF skips → optional, no reminder. Profile still marked complete.
10
Payment Details Setup
CONVERSATION
Agent guides new hire to set up bank details for payroll. Without this, they won't get paid.
Agent: "Let's set up your payment details so you get paid on time. I'll need your bank name, account number, and routing number."

IF employee provides all details → validate, store (encrypted, confidential)
IF employee skips → "I'll remind you before your first pay period. You won't receive payment without this."
IF first pay period approaching and still missing → daily reminder: "Your pay period ends Friday. I still need your bank details to process payment."
Security
Bank details encrypted and stored securely. Only the employee and payroll admin can view them.
11
IT Security Setup
AUTOMATIC
Agent guides the new hire through setting up two-factor authentication and any security requirements.
Agent: "One important step: let's set up two-factor authentication on your accounts. This keeps everything secure."

IF 2FA required by company policy → guide through setup, verify completion
IF VPN access needed (remote/hybrid) → provision VPN credentials and send setup guide
IF security key needed (admin roles) → add to equipment request
IF 2FA not set up within 24 hrs → reminder: "Your accounts won't be fully active until 2FA is enabled."
12
Training Assignments
AUTOMATIC
Agent assigns required trainings based on role. These are assigned on Day 1.
Trainings assigned based on role:
Security awareness (all employees), compliance training (all), tool-specific training (role-based), role-specific certifications (if any). List configurable by admin.
Agent: "I've assigned a few trainings for your first week: security awareness and [role-specific]. You can complete them at your own pace."

IF completed → logged
IF not completed within 7 days → reminder to employee
IF overdue by 14 days → escalate to manager
13
Team Introduction
AUTOMATIC
Agent posts a welcome message to the team so everyone knows who the new person is.
Team notification (Discord/Slack)
"Welcome Alex Chen! 🎉 Alex is joining as an intern starting today. He'll be working with the engineering team, reporting to Kunal. Fun fact: his superpower is [whatever they said]. Say hi!"
IF new hire completed fun facts → include in introduction
IF not completed yet → keep introduction simple (name, role, team)
14
Buddy/Mentor Assignment
AUTOMATIC
Agent assigns a buddy from the same team to help the new hire settle in during the first week.
IF buddy program configured by admin → assign buddy from same team, notify both
IF no buddy program → skip this step
Notifications
To buddy: "You've been paired with Alex Chen (new intern) for their first week. A quick intro and showing them around would be great."
To new hire: "I've paired you with [buddy name] for your first week. They'll help you get settled. Feel free to ask them anything."
15
First Week Agenda
AUTOMATIC
Agent shares a personalized first week plan so the new hire knows exactly what to expect.
Shared with new hire
"Here's your first week at AIntropy:

Day 1: Check-in with [manager] (already scheduled). Set up 2FA. Complete your profile.
Day 2: Team standup. Meet your buddy [name]. Start security training.
Day 3: 1-on-1 with [buddy]. Explore the codebase/docs.
Day 4: Continue trainings. Ask me anything you need.
Day 5: End-of-week check-in with [manager].

This is flexible. Just ask me if you want to adjust anything."
IF manager has shared specific first-week tasks → include them
IF not → use default agenda based on role
16
Equipment Assignment
NEEDS APPROVAL
Agent creates equipment request automatically based on role and work type.
Work type:
IF in-office → standard package (Engineer: MacBook Pro + monitor. Intern: MacBook Air. HR: MacBook Air)
IF remote → remote setup allowance ($1,500 full-time, $500 intern, customizable)
IF hybrid → in-office package + remote setup if needed

Budget threshold:
IF cost < $1,000 → send to manager for approval
IF cost ≥ $1,000 → send to manager AND finance for approval
IF non-standard item requested → flag: "Alex requested [item] instead of standard package. Approve?"
Admin sees in Kuri
"Alex Chen (intern, in-office) needs standard intern equipment package: MacBook Air ($999), basic peripherals ($200). Total: $1,199. Approve?"
[Approve] [Modify] [Reject]
IF approved → procurement processes order, log
IF modified → agent updates request, re-confirm with admin
IF rejected → notify employee with reason
IF no response in 48 hrs → reminder to admin
IF no response in 72 hrs → escalate to super admin
17
Schedule Manager Check-in (Day 1)
AUTOMATIC
Agent checks both the manager's and the new hire's calendar and schedules a 30-minute check-in on Day 1 itself.
Calendar invite
To: New hire + Reporting manager
Subject: "Welcome check-in: [Manager] + [New hire]"
Duration: 30 minutes
Note: "Quick hello and see if you need anything."
IF no common availability on Day 1 → suggest 3 nearest alternative slots and ask both to confirm

Monitor Completion
PROACTIVE
Agent checks progress every 4 hours and nudges the right person when something is stuck.
Checklist tracked:
□ Company email created
□ Policies acknowledged
□ GitHub access granted (if applicable)
□ Discord joined
□ All other registered tools provisioned
□ Profile created
□ Profile photo uploaded
□ Birthday added
□ Emergency contact added
□ Equipment assigned
□ Manager check-in scheduled
IF all items done → proceed to completion (Step 19)

IF stuck on same item for 24 hours → nudge the right person:
  Policy acknowledgement stuck → nudge employee
  Tool provisioning stuck → nudge admin/IT
  Profile incomplete → nudge employee
  Equipment not approved → nudge admin
  Manager check-in not scheduled → nudge manager

IF stuck for 48 hours → escalate to admin: "Alex Chen's onboarding is stuck. [Item] pending for 48 hours. Please intervene."

IF stuck for 72 hours → alert super admin

IF start date is tomorrow and not complete → URGENT alert: "Alex Chen starts tomorrow. Onboarding is [X]% complete. Blocking items: [list]. Please resolve today."

Error Handling
EDGE CASES
What can go wrong, and what the agent does about it.
HR tool webhook fails
Retry 3 times over 1 hour. If still fails, alert admin: "I detected a new contract on [tool] but couldn't pull the details. Please check."
Employee email already exists (possible rehire)
Agent checks: was this person previously offboarded? If yes, ask admin: "This appears to be a rehire. Reactivate existing accounts or create new ones?" If no, flag email conflict.
Employee doesn't log in within 3 days
Send reminder email. If no login in 5 days, alert manager: "Alex hasn't logged in since start date. Everything is set up on our end."
Manager leaves during onboarding
Agent detects manager is being offboarded. Alert admin: "Alex's reporting manager is leaving. Please assign a new manager before onboarding can complete."
Tool removed from registry during onboarding
Agent skips that tool and logs: "[Tool] was removed from registry. Skipping for Alex."
Multiple people onboarding at same time
Agent handles each independently. If same tool has rate limits, queue and process sequentially. Log any delays caused by rate limiting.
