PTO & Leave Workflow

From "I want Friday off" to calendar updated

1
Employee Requests Time Off TRIGGER
Employee says "I want Friday off" in free text via Kuri. No forms, no portals. Just a sentence.
2
Check Leave Balance AUTO
Agent checks the employee's current leave balance against the requested days.
IF balance ≥ requested days → proceed.
IF balance < requested → tell employee: "You only have X days left."
3
Check Team Calendar for Conflicts AUTO
Agent checks the team calendar to identify any scheduling conflicts.
IF no conflicts → proceed.
IF conflicts found → warn manager: "2 others are off the same week."
4
Prepare Request with Full Context AUTO
Agent assembles the complete request: dates, leave balance, conflict status, leave type, and any relevant notes for the manager.
5
Manager Approves APPROVAL NEEDED
Sent to manager with all context. Shows: "Shivangi wants Friday off. 12 days remaining. No team conflicts."
PTO is always a manager decision. The agent gathers context, the human decides.
6
Calendar Updated & Team Notified AUTO
After approval: calendar updated, team notified, leave balance adjusted automatically.
IF rejected → employee notified with the manager's reason.
7
Manager Reminder PROACTIVE
If manager hasn't responded in 24 hours, agent sends a gentle reminder to the manager.

One sentence from employee. One click from manager. Everything else is automatic.
